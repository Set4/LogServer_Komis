Клиент-серверное приложение.

Клиент-консольное приложение "LogSender".
Клиент требует введение имени которое будет отображаться в пакете отправляемом на сервер. 
После чего раз в 10 секунд устанавливает соединение с сервером по протоколу TCP 
отправлая пакет содержищий:нарастающий номер пакета, дату\время, введенное имя клмента. 
Получает от сервера ответ и отображает результат операции в консоли. 

Сервер-консольное приложение "Server".
Сервер прослушивает входящии соединения в отдельном потоке запускает обработчик, 
который получает данные и записывает во временный лог-файл в формате json(расположенный в папке Logs с запускаемым приложением, с именем "logfile.json"). 
Результат обработки (операции сохранения) - отправляется обратно клиенту. 
Ротацию логов сервера с периодом 1 минута происходит путем переименования временного лог-файла в файл с именем в формате "Log (dd/MM/yyyy H.mm.ss).json
